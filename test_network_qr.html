Network QR Code Test

Problem Identified

Issue: QR codes were encoding localhost:3000 which doesn't work when scanned from a phone.

Solution: Updated frontend/.env.development to use your computer's IP address: 192.168.1.111:3000

Changes Made

  1. Updated frontend/.env.development:
    • Changed: VITE_BACKEND_URL=http://localhost:3000
    • To: VITE_BACKEND_URL=http://192.168.1.111:3000
  2. Restarted frontend development server
  3. QR codes now encode: http://192.168.1.111:3000/track/[qrCodeId]

Test Instructions

  1. Open QR editor: http://localhost:5174
  2. Create a QR code with URL: www.expedia.com
  3. Click "Save to My QR codes"
  4. Check browser console for logs - look for tracking URL
  5. It should show: http://192.168.1.111:3000/track/[qrCodeId]
  6. Scan the QR code from your phone
  7. Check dashboard - scan count should now increase!

How It Works Now

  1. QR code encodes: http://192.168.1.111:3000/track/[qrCodeId]
  2. Phone scans QR code and visits that URL
  3. Backend records scan (increments count)
  4. Backend redirects phone to www.expedia.com
  5. Dashboard shows updated scan count

Verification Test

Test the backend directly:

# Test backend is accessible from network
curl http://192.168.1.111:3000/

# Should return: {"message":"StickerQR backend running","version":"1.0.0","environment":"development"}

Test from your phone:

  1. Open browser on phone
  2. Visit: http://192.168.1.111:3000/
  3. Should see the same JSON response

Troubleshooting

If it still doesn't work:

  1. Check Windows Firewall - allow port 3000
  2. Make sure backend is running: node backend/index.js
  3. Check MongoDB is running
  4. Verify QR code actually encodes the IP address (scan with QR scanner app)
  5. Check browser console for errors when saving QR code
