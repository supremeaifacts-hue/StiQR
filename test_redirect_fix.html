Test Redirect Fix for /track/:id Endpoint

Problem Fixed

Issue: The /track/:id endpoint was returning JSON instead of doing an HTML redirect.

Solution: Updated the endpoint to:

  1. Find the QR code in the database
  2. Increment the scan count
  3. Return a proper 302 redirect to the target URL

Changes Made to backend/index.js

// BEFORE:
res.redirect(`/api/assets/qrcodes/${id}/redirect`);

// AFTER:
// Increment scan count
qrCode.scans = (qrCode.scans || 0) + 1;
qrCode.lastScanned = new Date();
user.stats.totalScans = (user.stats.totalScans || 0) + 1;
await user.save();

// Redirect to target URL
let targetUrl = qrCode.data;
if (!targetUrl.startsWith('http://') && !targetUrl.startsWith('https://')) {
  targetUrl = 'https://' + targetUrl;
}
return res.redirect(targetUrl);

Test the Fix

Follow these steps:

  1. Create a QR code in the editor (e.g., for google.com)
  2. Save it to your dashboard
  3. Get the QR code ID from browser console or dashboard
  4. Test the tracking URL directly

Test URL format: http://192.168.1.111:3000/track/[qrCodeId]

Test with dummy ID: http://192.168.1.111:3000/track/test123

Note: Replace test123 with an actual QR code ID from your dashboard

Expected Behavior

  1. When you visit /track/[qrCodeId]:
  2. Backend finds the QR code
  3. Scan count increments by 1
  4. Browser receives 302 redirect
  5. Browser automatically follows redirect to target URL (e.g., google.com)
  6. Dashboard shows updated scan count
