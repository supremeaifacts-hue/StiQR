2. Routing Issue

Problem: /track/:id route was mounted at /api/track/:id

Solution: Added root-level route in backend/index.js

// Added to backend/index.js app.get('/track/:id', async (req, res) => { // ... redirects to /api/assets/qrcodes/${id}/redirect });

Why This Happens:

  1. QR code generated but not saved: You might have generated a QR code but didn't click "Save to My QR codes"
  2. ID mismatch: Frontend generates ID, backend might generate different ID if not passed correctly
  3. Scanning wrong QR code: You might be scanning an old/incorrect QR code
